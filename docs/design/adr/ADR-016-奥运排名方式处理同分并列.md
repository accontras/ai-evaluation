## ADR-016：奥运排名方式处理同分并列

| 项目 | 内容 |
|---|---|
| **上下文** | 跨对象排序时，多个对象可能总分相同。需要决定并列时的排名方式。 |
| **决策** | 采用奥运排名方式（1,1,3,4...），同分对象并列同一排名，后续排名跳过。 |
| **备选方案** | ① 密集排名（1,1,2,3...）；② 按额外字段（如 bizId 字母序）打破并列 |
| **影响** | 符合业务直觉，与体育赛事排名一致；`rank_total` 在排序时顺手写入，避免前端每次多查 count |
