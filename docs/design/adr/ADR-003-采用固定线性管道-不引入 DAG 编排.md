## ADR-003：采用固定线性管道，不引入 DAG 编排

| 项目 | 内容 |
|---|---|
| **上下文** | 评估流程的步骤（加载配置→获取数据→计算得分→汇总结果）是固定的业务逻辑，不存在用户可编排、动态增减步骤的需求。 |
| **决策** | 采用固定线性管道模型，Handler 顺序硬编码在 `ConfigurablePipeline.init()` 中。 |
| **备选方案** | ① DAG 配置表驱动（步骤可由业务人员编排）；② DAG 引擎（步骤有分支/并行/条件跳转） |
| **影响** | 实现简单、可读性高；如果未来需要动态编排步骤，需要重构为配置驱动模型 |
