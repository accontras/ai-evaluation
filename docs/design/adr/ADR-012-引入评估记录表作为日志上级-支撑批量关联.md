## ADR-012：引入评估记录表作为日志上级，支撑批量关联

| 项目 | 内容 |
|---|---|
| **上下文** | V1.2 中日志表（dr_index_log_base）是独立记录，批量评估时无法将多个对象的评估结果关联到同一次任务。 |
| **决策** | 新增 `dr_evaluation_record_base` 评估记录表，一次评估任务对应一条记录，日志表通过 `record_id` 关联。批量排序、状态汇总均基于 record_id 查询。 |
| **备选方案** | ① 用 batchNo 字符串关联（无独立表）；② 在日志表上加 batch_id 字段 |
| **影响** | 批量评估有明确的任务边界，支持状态追踪和排序；代价是多一张表和一次写入，但收益远大于成本 |
